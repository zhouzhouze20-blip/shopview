## 详细启动步骤

### 第一步：准备环境

#### 1.1 激活虚拟环境（推荐）
```bash
cd /Users/zhou/Library/Containers/com.tencent.xinWeChat/Data/Documents/xwechat_files/zhouzeze2011_6ba3/msg/file/2026-01/ShopView
source .venv/bin/activate
```

若尚未创建虚拟环境，可先创建再安装依赖：
```bash
python3 -m venv .venv
source .venv/bin/activate
```

#### 1.2 安装依赖（首次运行必做）
**未安装依赖时运行 `start-manual.py` 会报错 `ModuleNotFoundError: No module named 'sqlalchemy'`，请先执行：**
```bash
# 在项目根目录下，先激活 .venv 再安装
pip install -r python_requirements.txt

# 安装前端依赖
cd client
npm install
cd ..
```

#### 1.3 配置环境变量
确保 `env.app` 文件配置正确：
```bash
# 检查配置文件
cat env.app
```

主要配置项：
- `PGHOST`: 数据库主机地址
- `PGPORT`: 数据库端口（默认 5432）
- `PGDATABASE`: 数据库名称
- `PGUSER`: 数据库用户名
- `PGPASSWORD`: 数据库密码
- `PORT`: 后端服务端口（默认 7000）

当前已配置为：
```env
DATABASE_URL=postgresql://sales_user:sales_password_2024@192.168.98.80:5432/sales_db
PGHOST=192.168.98.80
PGPORT=5432
PGUSER=sales_user
PGPASSWORD=sales_password_2024
PGDATABASE=sales_db
```

```bash
# 检查 7000 端口（后端）

lsof -i:5173

# 释放端口

```

```

### 数据库迁移（Alembic）

**必须先进入项目根目录**，虚拟环境在项目根目录下名为 **`.venv`**。

```bash
# 1. 进入项目根目录
cd /Users/zhou/Library/Containers/com.tencent.xinWeChat/Data/Documents/xwechat_files/zhouzeze2011_6ba3/msg/file/2026-01/ShopView

# 2. 进入 python_app 并执行迁移（用 .venv 里的 Python，避免 command not found）
cd python_app
./alembic_upgrade.sh
```

`alembic_upgrade.sh` 会自动尝试 `../venv` 或 `../.venv`，若你用自定义虚拟环境，可设置 `ALEMBIC_PYTHON`：
```bash
ALEMBIC_PYTHON=/path/to/python ./alembic_upgrade.sh
```

**若报错 `Can't locate revision identified by 'initial_shopview'`**：说明数据库里记的版本是 `initial_shopview`，但本机没有对应迁移文件，连 `alembic stamp` 也会报同样错。用脚本直接改数据库里的版本号（不经过 Alembic 加载迁移），再执行升级。在 `python_app` 目录下执行：

  ```bash
  ../.venv/bin/python scripts_stamp_replace_floors.py
  ./alembic_upgrade.sh
  ```

## 故障排查

### 问题 0: alembic: command not found / .venv 或 venv 找不到

**症状**：执行 `alembic upgrade head` 提示 `command not found`，或 `../.venv/bin/python` 提示 `no such file or directory`。

**原因**：未在项目根目录，或本机虚拟环境目录名是 `venv` 不是 `.venv`。

**解决方法**：
1. 先进入**项目根目录**（包含 `python_app` 和虚拟环境的那一层）：
   ```bash
   cd /Users/zhou/Library/Containers/com.tencent.xinWeChat/Data/Documents/xwechat_files/zhouzeze2011_6ba3/msg/file/2026-01/ShopView
   ```
2. 再进入 `python_app`，用**项目根目录下的**虚拟环境运行 alembic：
   ```bash
   cd python_app
   ./alembic_upgrade.sh
   ```
   脚本会自动识别 `../.venv` 或 `../venv`；需要自定义解释器时设置 `ALEMBIC_PYTHON`。

### 问题 1: 端口被占用

**症状**：`ERROR: [Errno 48] Address already in use`

**解决方法**：
```bash
# 释放后端端口
kill -9 $(lsof -ti:7000)

# 释放前端端口
kill -9 $(lsof -ti:5173)
```

---

### 问题 2: 数据库连接失败

**症状**：`psycopg2.OperationalError: connection to server failed`

**解决方法**：
1. 检查数据库配置：
   ```bash
   cat env.app
   ```

2. 测试数据库连接：
   ```bash
   python3 test-db-simple.py
   ```

3. 检查网络连接：
   ```bash
   ping 192.168.98.80  # 替换为你的数据库地址
   ```

4. 如果无法连接数据库，可以使用 Mock API（仅用于演示）：
   ```bash
   ./start-mock-api.sh
   ```

---

### 问题 3: 模块导入错误

**症状**：`ModuleNotFoundError: No module named 'models'`

**解决方法**：
- 确保在项目根目录运行启动脚本
- 或者使用 `start-manual.py` 脚本（已修复路径问题）

---

### 问题 4: 前端依赖缺失

**症状**：`Error: The following dependencies are imported but could not be resolved`

**解决方法**：
```bash
cd client
rm -rf node_modules package-lock.json
npm install
```

---

### 问题 5: Python 依赖缺失

**症状**：`ModuleNotFoundError: No module named 'fastapi'`

**解决方法**：
```bash
pip install -r python_requirements.txt
```

## 提示

1. **首次启动**：确保已安装所有依赖
2. **端口冲突**：如果端口被占用，使用 `kill-port-7000.sh` 或相应脚本释放
3. **数据库连接**：如果无法连接，先运行 `test-db-simple.py` 测试
4. **代码修改**：修改代码后需要重启服务（当前 reload 已禁用）
5. **日志查看**：启动信息会显示在终端，错误信息也会显示在这里
