# 活动分析模块代办

## 一期口径

- [x] 一期只分析卡券使用，不展开满减、折扣、赠品、银行卡等其他活动规则。
- [x] 暂不建设活动规则明细表，先以 `tktpopinfo` 活动档期和卡券相关字段为准。
- [x] 暂不建设活动归因关系表，先按卡券使用记录、小票记录和活动档期做分析。
- [x] 暂不建设活动成本表，成本和毛利直接取 `salegoodslist`。
- [x] 组织、柜组、部门维度直接使用现有组织柜组部门表。
- [x] 商品、品牌、品类维度直接使用现有商品相关表。
- [x] 会员信息取 `fj_dw_member_dim` 实时信息，暂不做会员分层快照。
- [x] 暂不建设活动前后对比汇总表，第一版先做活动/卡券结果分析。

## 已具备的数据表

- [x] 会员维表：`fj_dw_member_dim`
- [x] 卡券返券日志表：`tktcardfqlog`
- [x] 活动档期表：`tktpopinfo`
- [x] 销售主单：`salehead`
- [x] 销售商品明细：`salegoods`
- [x] 销售付款明细：`salepay`
- [x] 销售单品日汇总：`salegoodslist`
- [x] 组织/柜组/部门表：沿用现有表
- [x] 商品/品牌/品类表：沿用现有表

## 一期重点校验

- [ ] 校验卡券日志表 `tktcardfqlog` 中是否同时覆盖发券、用券、冲正/退券等业务动作。
- [ ] 梳理 `tktcardfqlog.tcflzy` 的摘要取值，明确哪些代表发券、哪些代表用券、哪些代表冲正。
- [ ] 梳理 `tktcardfqlog.tcflsource` 的券来源取值，确认销售返券、买券、退货返券、后台新增等分类。
- [ ] 校验 `tktcardfqlog.tcflpopid` 是否能关联 `tktpopinfo.tpiid`。
- [ ] 校验 `tktcardfqlog.tcflsyjid + tcflinvno + tcflmkt` 是否能关联销售小票。
- [ ] 校验 `tktcardfqlog.tcflsyjtrace` 是否能辅助关联小票交易流水。
- [ ] 校验 `salepay` 中 `paycode = '0500'` 和 `paycode = '0580'` 的付款记录是否都属于卡券使用。
- [ ] 校验 `salepay.payno`、`coupon_trace_seqno`、`coupon_event_id`、`coupon_policy_id` 与 `tktcardfqlog` 的对应关系。
- [ ] 校验 `salehead.ybillno`、`salehead.ysyjh`、`salehead.yfphm` 是否能识别退货对应原小票。

## 一期可直接产出的指标

- [ ] 活动档期清单：活动编码、活动主题、活动开始日期、活动结束日期。
- [ ] 卡券发生金额：按活动、日期、门店、会员、券号汇总 `tktcardfqlog.tcflmoney`。
- [ ] 卡券余额变化：按会员、券号、活动看 `tktcardfqlog.tcflye`。
- [ ] 卡券付款金额：按小票、活动、付款方式汇总 `salepay.je`。
- [ ] 0500/0580 付款金额：单独统计两类卡券付款方式。
- [ ] 活动销售额：关联小票后统计 `salehead` / `salegoods` / `salegoodslist` 销售金额。
- [ ] 活动成本和毛利：直接取 `salegoodslist` 中销售成本、毛利、净毛利相关字段。
- [ ] 部门/柜组贡献：通过销售表关联现有组织柜组部门表。
- [ ] 商品/品牌/品类贡献：通过销售表关联现有商品表。
- [ ] 会员贡献：通过会员号关联 `fj_dw_member_dim` 实时会员等级、注册渠道、状态等信息。
- [ ] 退货冲减：通过销售表原小票字段识别退货并冲减活动结果。

## 一期页面/模块代办

- [ ] 活动列表
  - 展示 `tktpopinfo` 活动档期。
  - 支持按活动编码、活动主题、日期筛选。

- [ ] 活动总览
  - 展示销售额、订单数、消费人数、会员人数、卡券付款金额、退货金额、净销售额、毛利、净毛利。
  - 成本和毛利取 `salegoodslist`，不单独维护成本表。

- [ ] 卡券使用分析
  - 展示卡券发放/使用/冲正流水。
  - 同时展示 `tktcardfqlog` 口径和 `salepay paycode in ('0500', '0580')` 口径。
  - 输出两边差异：有卡券日志无付款、有付款无卡券日志、金额不一致。

- [ ] 部门/柜组分析
  - 按现有组织柜组部门表汇总销售、用券金额、毛利、净毛利、退货冲减。
  - 输出部门排行和异常部门。

- [ ] 商品/品牌/品类分析
  - 按现有商品表汇总销售、用券金额、毛利、净毛利、退货冲减。
  - 输出品牌、品类、商品排行。

- [ ] 会员分析
  - 使用 `fj_dw_member_dim` 实时会员信息。
  - 统计会员等级、注册渠道、会员状态、年龄段、性别等维度下的活动贡献。

- [ ] 数据质量检查
  - 活动ID无法关联。
  - 卡券日志无法关联小票。
  - 0500/0580付款无法关联卡券日志。
  - 小票无会员号。
  - 销售明细缺失柜组/商品映射。
  - 退货原单无法识别。
