## 还需要补充或确认的数据

- [ ] 活动规则明细表
  - 用途：区分满减、返券、折扣、赠品、银行卡活动等具体规则。
  - 关键字段：活动ID、规则ID、规则名称、规则类型、门槛金额、返券/优惠金额、适用部门、适用品牌、适用商品、是否互斥、是否叠加。
  - 当前缺口：`tktpopinfo` 只有活动档期，没有完整活动规则。

- [ ] 卡券核销/使用明细表
  - 用途：明确每张券什么时候被谁用了、用在哪张小票上。
  - 关键字段：券号、会员号、活动ID、规则ID、核销时间、门店、收银机号、小票号、交易流水号、用券金额、券面值、券状态。
  - 当前可替代字段：`salepay.coupon_trace_seqno`、`coupon_event_id`、`coupon_policy_id`、`payno`，但仍需确认能否稳定对应单张券。

- [ ] 活动与销售小票归因关系表
  - 用途：避免只按日期归因，解决活动1-10号、券5-15号、11号退货这类问题。
  - 建议产出表：`fact_activity_order_attribution`
  - 关键字段：活动ID、小票单据号、归因类型、交易时间、是否活动期、是否券后效、是否退货冲减、原小票单据号、归因金额、归因成本。

- [ ] 活动售后观察口径表
  - 用途：记录活动结束后7/15/30天内退货、撤销、冲正。
  - 建议产出表：`fact_activity_after_sale`
  - 关键字段：活动ID、原小票单据号、退货小票单据号、退货时间、退货金额、退货商品、退货会员、冲减口径。

- [ ] 活动成本表
  - 用途：计算净销售、净毛利、ROI。
  - 关键字段：活动ID、成本类型、成本金额、承担方、部门、品牌、供应商、发生日期。
  - 成本类型建议：门店承担券损、供应商承担券损、企划费用、广告费用、物料费用、赠品成本、人工费用。

- [ ] 组织/柜组/部门映射表
  - 用途：把 `salegoods.gz`、`salegoodslist.sglmfid`、会员表里的 `depart_name2` 统一到经营部门、楼层、品类、厅位。
  - 可参考现有表：`manaframe`、柜组/经营单元相关表。
  - 当前风险：如果编码和中文部门名称不能统一，部门分析会出现口径不一致。

- [ ] 商品/品牌/品类维表确认
  - 用途：按商品、品牌、品类分析活动效果。
  - 可参考现有表：`goodsbase`、`goodsbarcode`、`goodsmframe`、`suppliergoods`。
  - 需要确认：品牌编码、品类编码、柜组编码是否能和 `salegoods`、`salegoodslist` 稳定关联。

- [ ] 会员分层快照表
  - 用途：固定活动开始前的会员身份，避免后续等级变化影响复盘。
  - 建议产出表：`snap_member_segment`
  - 关键字段：快照日期、会员号、会员等级、新老客标识、核心会员标识、高价值会员标识、近30/90/180天消费金额、消费次数、最近消费日期。

- [ ] 活动前后对比汇总表
  - 用途：支持活动前7/14天、活动期间、活动后7/14/30天对比。
  - 建议产出表：`fact_activity_period_compare`
  - 关键字段：活动ID、时间窗口、部门、品牌、会员类型、销售额、订单数、消费人数、客单价、人均贡献、毛利、退货金额、净销售额。
