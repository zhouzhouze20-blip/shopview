# 收益地图可视化模块代办

目标：基于合同底图/柜位图，按财务月展示每个铺位收益。收益由三部分组成：销售毛利、费用收取、财务补录额外收费。

## 1. 口径确认

- [ ] 确认财务月规则：每个财务月的开始日期、结束日期，以及跨月数据归属方式。
- [ ] 确认销售毛利口径：是否使用销售表现有 `net_profit = SUM(sgln2)`。
- [ ] 确认费用收入口径：结算单费用取 `supsetcharge.sscmoney`，按结算单日期、结算期间还是付款日期归属财务月。
- [ ] 确认额外收费口径：供应商直接打款给财务后，由财务后台补录，只有确认状态才进入收益。
- [ ] 确认费用正负方向：扣款、返款、冲销是否需要区分正负。
- [ ] 确认一个柜组对应多个铺位时的分摊规则：主铺位归属、按面积分摊、按人工比例分摊，或暂不分摊。
- [ ] 确认收益地图默认展示指标：总收益、销售毛利、费用收取、额外收费。

## 2. 数据关系梳理

- [ ] 梳理 `business_units` 铺位数据是否覆盖需要展示的所有合同底图柜位。
- [ ] 梳理 `geo_elements` 当前生效柜位图版本是否能正确关联到 `business_units.id`。
- [ ] 梳理 `business_unit_binding` 中铺位、合同、柜组、供应商关系是否完整。
- [ ] 输出异常清单：有铺位无合同/柜组、有柜组收益无铺位、一个柜组绑定多个铺位、一个铺位绑定多个有效合同。
- [ ] 确认合同有效期与财务月的匹配规则，避免历史合同和当前合同混算。

## 3. 数据库设计

- [ ] 新增财务补录表 `revenue_extra_receipts`。
- [ ] 字段包含：门店、楼层、铺位、财务月、合同号、供应商、柜组、金额、到账日期、凭证号、收费类型、备注、状态、创建人、确认人。
- [ ] 补录状态建议：`DRAFT`、`CONFIRMED`、`VOID`。
- [ ] 默认收益统计只纳入 `CONFIRMED` 数据。
- [ ] 评估是否新增月收益快照表 `unit_monthly_revenue_snapshot`。
- [ ] 如果新增快照表，字段包含：财务月、铺位、合同、供应商、柜组、销售毛利、费用收取、额外收费、总收益、生成时间。
- [ ] 增加必要索引：`financial_month`、`unit_id`、`contract_id`、`supplier_id`、`counter_group_id`。

## 4. 后端接口

- [ ] 新增收益地图聚合接口：`GET /api/revenue-map/monthly`。
- [ ] 聚合接口支持筛选：门店、楼层、财务月、柜位图版本、展示指标。
- [ ] 聚合接口返回：底图信息、柜位图形、铺位收益、异常标记。
- [ ] 新增铺位收益详情接口：`GET /api/revenue-map/units/{unit_id}/detail`。
- [ ] 详情接口返回：销售毛利来源、结算费用来源、财务补录来源、合计金额。
- [ ] 新增财务补录接口：创建、修改、确认、作废、列表查询。
- [ ] 增加重算接口：按财务月重算收益快照。
- [ ] 接入权限：`revenue.view`、`revenue.extra.create`、`revenue.extra.confirm`、`revenue.extra.void`。

## 5. 前端页面

- [ ] 新增页面：收益地图。
- [ ] 顶部筛选：门店、楼层、财务月、柜位图版本、收益指标。
- [ ] 主区域复用合同底图和柜位图 SVG 叠加能力。
- [ ] 根据收益金额给铺位着色，支持总收益和三项分项切换。
- [ ] 无数据铺位显示灰色。
- [ ] 负收益或异常铺位增加明显描边。
- [ ] 有财务补录的铺位增加标识。
- [ ] 鼠标悬浮展示铺位号、品牌/供应商、合同号、三项收益和总收益。
- [ ] 点击铺位打开详情抽屉，展示来源明细和近 6-12 个财务月趋势。
- [ ] 增加排行和异常列表：收益 TOP、低收益、负收益、补录铺位、未匹配收益。

## 6. 财务补录后台

- [ ] 增加财务补录列表页或弹窗入口。
- [ ] 支持按财务月、供应商、合同号、铺位查询。
- [ ] 支持录入金额、到账日期、凭证号、收费类型、备注。
- [ ] 支持草稿保存。
- [ ] 支持财务确认，确认后进入收益地图。
- [ ] 支持作废，作废后从收益统计剔除。
- [ ] 保留创建人、修改人、确认人和时间。

## 7. 验收与对账

- [ ] 选取一个财务月和一个楼层做样例验收。
- [ ] 校验销售毛利合计与销售看板同口径数据一致。
- [ ] 校验费用收取合计与联营结算单费用合计一致。
- [ ] 校验额外收费合计与财务补录列表一致。
- [ ] 校验铺位地图总收益等于三项收益之和。
- [ ] 校验异常清单是否能覆盖未绑定、重复绑定、多铺位分摊等情况。
- [ ] 校验不同权限用户只能看到授权范围内的铺位和收益。
